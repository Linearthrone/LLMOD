# Task · Android companion MVP v2 (audio + reliability)

## Context

`AndroidRemoteCompanion/` MVP exists with health + text chat. Product direction remains:

- PC is brain/home
- Unreal is embodiment
- phone is remote communication surface

Current gap to roadmap M4: no `/chat-audio` capture/upload yet, and no in-app conversation log/history view.

## Scope

1. **Add audio upload path** in Android app
   - Record short mic clip and call `POST /api/remote/v1/chat-audio` (multipart field `audio`)
   - Display reply + `conversationId`
   - Handle key error responses (`audio_field_required`, `multipart_form_required`, `unauthorized`)

2. **Conversation UX upgrade**
   - Keep a local chronological message list (user + assistant) in the app session
   - Preserve base URL/token/contactId settings

3. **Safety / robustness**
   - Validate URL/token before send
   - Add clear loading/error states and retry for network failures

## Out of scope

- Video streaming
- Full push notifications
- End-to-end UE animation logic

## Acceptance criteria

- [ ] Android app can send text and audio to the existing House Victoria remote API.
- [ ] Manual test evidence included for both `/chat` and `/chat-audio`.
- [ ] `AndroidRemoteCompanion/README.md` updated with audio usage and permissions notes.
