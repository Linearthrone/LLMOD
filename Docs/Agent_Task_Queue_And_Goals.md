# Agent task queue and team goals

House Victoria uses a small file-based queue for **PM / DEV / OPS / QA** work, plus a single **strategic goals** document for the remote companion and “AI at home” direction.

**Canonical location (this repo: `Docs/agents/`):**

| What | Path |
|------|------|
| **Strategic goals** (phases, milestones, guardrails) | [`agents/GOALS-Remote-Companion-and-AI-Home.md`](agents/GOALS-Remote-Companion-and-AI-Home.md) |
| **How the queue works** | [`agents/README.md`](agents/README.md) |
| **Open task assignments** | [`agents/tasks/`](agents/tasks/) |
| **Completion reports** | [`agents/reports/`](agents/reports/) |
| **Archived (closed task + report pairs)** | [`agents/log/`](agents/log/) |

**Role handbooks** (long-form, not the live task list) stay under [`agents/agents/`](agents/agents/) (e.g. `PM-01.md`, `DEV-01.md`).

For day-to-day product and technical context, start with [HouseVictoria_Guide.md](HouseVictoria_Guide.md).
